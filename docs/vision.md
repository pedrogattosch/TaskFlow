# Visão — TaskFlow

## 1. Visão geral

O TaskFlow será um sistema web para cadastro, organização, acompanhamento e conclusão de tarefas.

A proposta do projeto é ser simples o suficiente para ser desenvolvido por uma pessoa em fase de aprendizado, mas estruturado de forma profissional, demonstrando boas práticas de arquitetura, organização e desenvolvimento full stack.

---

## 2. Objetivo do projeto

O objetivo do TaskFlow é permitir que usuários gerenciem suas tarefas de forma clara, organizada e eficiente, acompanhando todo o ciclo de vida de cada tarefa.

Além disso, o projeto tem como objetivo servir como portfólio técnico, evidenciando:

- domínio de desenvolvimento com .NET
- integração com banco SQL Server
- construção de APIs REST
- consumo de API com React
- organização com Clean Architecture
- boas práticas de código e versionamento

---

## 3. Público-alvo

Usuários que desejam organizar tarefas pessoais de forma simples, com controle de:

- status
- prioridade
- prazos
- categorias

Inicialmente, o sistema será voltado para uso individual.

---

## 4. Escopo da versão 1.0

Funcionalidades principais:

- cadastro de usuário
- login e autenticação com JWT
- criação de tarefas
- edição de tarefas
- exclusão lógica
- listagem com filtros
- ordenação
- alteração de status
- dashboard com resumo por status
- categorização de tarefas

---

## 5. Funcionalidades futuras

- etiquetas múltiplas
- subtarefas
- comentários
- compartilhamento
- notificações
- tema claro/escuro

---

## 6. Problema resolvido

Centralizar o gerenciamento de tarefas em um único sistema simples e eficiente, substituindo métodos dispersos como anotações ou apps limitados.

---

## 7. Valor entregue

- organização pessoal
- controle de prioridades
- acompanhamento de progresso
- visualização clara de tarefas

---

## 8. Stack tecnológica

- Back-end: .NET
- Front-end: React
- Banco: SQL Server
- Autenticação: JWT

---

## 9. Critérios de sucesso

- fluxo completo de tarefas funcionando
- autenticação implementada
- API funcional e documentada
- front-end integrado
- projeto executável localmente
- código organizado e versionado
